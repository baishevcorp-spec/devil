# Devil — Интеллектуальный агент-ассистент для VS Code

**Devil** — это расширение для VS Code, представляющее собой интеллектуального агента-ассистента для разработчиков с долговременной памятью. Агент решает фундаментальную проблему потери контекста проекта на протяжении всего жизненного цикла разработки.

##  Возможности

### Основные функции

- ** Чат-интерфейс** — общение с агентом на естественном языке прямо в VS Code
- **🧠 Долговременная память** — графовая база знаний хранит структуру проекта, зависимости, архитектурные решения
- **📋 Генерация артефактов** — автоматическое создание Roadmap, чек-листов, планов разработки
- **🔍 Анализ кода** — объяснение кода, рефакторинг, поиск зависимостей, генерация тестов
- **🤖 Мульти-модельность** — поддержка 7+ LLM моделей (GPT-4o, Claude, Gemini и др.)
- **⚡ Быстрый поиск** — полнотекстовый и семантический поиск по проекту
- **🔗 Git-интеграция** — история коммитов, diff между версиями, анализ изменений

### Ключевые команды

| Команда | Описание |
|---------|----------|
| `/explain` | Объяснить код файла или выделенного фрагмента |
| `/refactor` | Предложить рефакторинг кода (SOLID, паттерны) |
| `/roadmap generate` | Сгенерировать Roadmap проекта на основе интервью |
| `/checklist generate` | Создать чек-лист файлов на основе Roadmap |
| `/dev generate` | Сгенерировать план разработки с пошаговым выполнением |
| `/search` | Полнотекстовый поиск по проекту |
| `/whereis` | Найти все использования символа в проекте |
| `/test generate` | Сгенерировать юнит-тесты для файла |
| `/lint` | Запустить линтер и показать отчёт |
| `/memory show` | Показать графовую память проекта |

##  Установка

### Требования

- **VS Code** 1.80 или выше
- **Node.js** 18+ (для разработки)
- **Windows 11** (основная платформа)
- **API-ключ** прокси-сервиса (MyProxyapi.ru или другой OpenAI-совместимый)

### Установка из .vsix пакета

1. Скачайте последний релиз с [GitHub Releases](https://github.com/yourusername/devil/releases)
2. Откройте VS Code
3. Перейдите в раздел расширений (`Ctrl+Shift+X`)
4. Нажмите `...` → `Install from VSIX...`
5. Выберите скачанный файл `devil-0.5.0.vsix`
6. Перезапустите VS Code

### Установка из исходников (для разработчиков)

```bash
# Клонируйте репозиторий
git clone https://github.com/yourusername/devil.git
cd devil

# Установите зависимости
npm install

# Соберите расширение
npm run compile

# Запустите в режиме разработки
F5 (в VS Code)
```

## ⚙️ Настройка

### 1. Настройка прокси и API-ключа

Откройте настройки VS Code (`Ctrl+,`) и добавьте:

```json
{
  "devil.baseUrl": "https://api.proxyapi.ru/openai/v1",
  "devil.apiKey": "your-api-key-here",
  "devil.model": "gpt-4o-mini"
}
```

**Поддерживаемые прокси-сервисы:**
- MyProxyapi.ru (рекомендуется)
- OpenAI API (прямое подключение)
- Ollama (локальные модели)
- Любой OpenAI-совместимый API

### 2. Выбор модели

Используйте команду `/model switch` для переключения между моделями:

- `gpt-4o-mini` — быстрая модель для чата
- `gpt-4o` — мощная модель для сложных задач
- `gpt-5.5` — флагманская модель OpenAI
- `claude-sonnet-4` — альтернатива от Anthropic
- `gemini-2.0-flash` — модель от Google
- `o4-mini` — модель для рассуждений
- `gpt-5.5-pro` — максимальная мощность

### 3. Настройка системного промпта

Вы можете задать кастомный системный промпт для агента:

```json
{
  "devil.systemPrompt": "Ты — опытный TypeScript разработчик. Всегда используй строгую типизацию и следуй принципам SOLID."
}
```

##  Использование

### Начало работы с новым проектом

1. **Откройте проект** в VS Code
2. **Запустите чат** командой `Devil: Open Chat` (`Ctrl+Shift+P`)
3. **Откройте папку проекта** командой `Devil: Open Project`
4. **Сгенерируйте Roadmap**:
   ```
   /roadmap generate
   ```
   - Для пустого проекта агент проведёт интервью (создаст `interview.json`)
   - Заполните файл и выполните команду снова
   - Roadmap будет сохранён в `.devil/roadmap.md`

5. **Создайте чек-лист**:
   ```
   /checklist generate
   ```
   Чек-лист будет основан на Roadmap и структуре проекта.

6. **Сгенерируйте план разработки**:
   ```
   /dev generate
   ```
   Агент создаст пошаговый план с зависимостями между файлами.

### Пошаговая разработка

После генерации плана используйте команды:

```bash
/dev next      # Выполнить следующий шаг (создать файл/директорию)
/dev status    # Показать прогресс выполнения
/dev skip      # Пропустить шаг
/dev reset     # Сбросить план и начать заново
```

### Анализ существующего кода

```bash
/explain src/services/UserService.ts    # Объяснить файл
/refactor src/utils/helpers.ts          # Предложить рефакторинг
/whereis App                            # Найти все использования класса App
/search useEffect                       # Полнотекстовый поиск
```

### Работа с Git

```bash
/git log src/extension.ts               # История коммитов файла
/diff abc123 def456                     # Diff между коммитами
```

### Управление памятью

```bash
/memory show                            # Показать граф памяти
/memory add "React используется для UI" # Добавить знание
/memory delete <id>                     # Удалить узел
```

## 🏗️ Архитектура

Devil построен по принципу разделения ответственности и состоит из 6 основных модулей:

```
┌─────────────────────────────────────────────────────────────┐
│                     Frontend (Webview)                       │
│                  Chat UI, Markdown рендеринг                 │
─────────────────────────────────────────────────────────────┘
                              ↕ postMessage
┌─────────────────────────────────────────────────────────────┐
│                   Backend (Extension Host)                   │
│                                                              │
│  ┌──────────────┐  ┌──────────────┐  ┌──────────────────┐  │
│  │   Core       │  │   Project    │  │      LLM         │  │
│  │  Layer       │  │   Layer      │  │      Layer       │  │
│  │              │  │              │  │                  │  │
│  │ Extension    │  │ Project      │  │ LLM Provider     │  │
│  │ Entry        │  │ Manager      │  │ Multi-Model      │  │
│  │ Command      │  │ File System  │  │ Manager          │  │
│  │ Handler      │  │ Context      │  │                  │  │
│  │ Config       │  │ Builder      │  │                  │  │
│  └──────────────┘  └──────────────┘  └──────────────────┘  │
│                                                              │
│  ┌──────────────┐  ┌────────────────────────────────────┐  │
│  │   Memory     │  │         Analysis Layer             │  │
│  │   Layer      │  │                                    │  │
│  │              │  │ Code Analyzer  Git Service         │  │
│  │ Memory Store │  │ Search Index   Graph Builder       │  │
│  │ Graph Builder│  │                                    │  │
│  │ User Profile │  └────────────────────────────────────┘  │
│  └──────────────┘                                           │
└─────────────────────────────────────────────────────────────┘
```

**Подробная документация:** [ARCHITECTURE.md](docs/ARCHITECTURE.md)

### Технологический стек

| Компонент | Технология |
|-----------|-----------|
| Расширение VS Code | TypeScript, VS Code API |
| Backend-логика | Node.js (TypeScript) |
| Хранилище данных | SQLite (графовая память, кэш) |
| HTTP-клиент | Axios |
| Парсинг кода | Tree-sitter (WASM) |
| Полнотекстовый поиск | flexsearch |
| Markdown-рендеринг | marked + highlight.js |
| Тестирование | Jest + ts-jest |
