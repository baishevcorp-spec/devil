## 📋 План реализации (поэтапно)

### Этап 1: Подготовка и установка зависимостей (~30 мин)

1. Установить `transformers.js`:
   ```bash
   npm install @xenova/transformers
   ```

2. Обновить `package.json` (добавить в dependencies)

3. Создать миграцию 003 для таблицы `node_embeddings`

### Этап 2: Расширение схемы БД (~1 час)

1. Обновить `src/services/MemoryStore.ts`:
   - Добавить метод `createNodeEmbeddingsTable()`
   - Добавить методы для работы с embeddings:
     - `saveNodeEmbedding(nodeId, embedding, model, dimensions, textHash)`
     - `getNodeEmbedding(nodeId)`
     - `deleteNodeEmbedding(nodeId)`
     - `findNodesWithoutEmbeddings()`

2. Обновить `src/interfaces/IMemoryStore.ts`:
   - Добавить интерфейс `NodeEmbedding`
   - Добавить интерфейс `MemorySearchResult`
   - Расширить `NodeType` типом `'reference'`

3. Применить миграцию 003 при инициализации

### Этап 3: Реализация векторизации (~2 часа)

1. Создать `src/services/EmbeddingService.ts`:
   - Загрузка модели `Xenova/all-MiniLM-L6-v2` (384 dimensions)
   - Метод `generateEmbedding(text: string): Promise<Float32Array>`
   - Метод `cosineSimilarity(a: Float32Array, b: Float32Array): number`
   - Кэширование модели (загрузка один раз)

2. Обновить `src/services/SearchIndex.ts`:
   - Добавить метод `buildNodeEmbeddings()` — векторизация всех узлов без embeddings
   - Добавить метод `updateNodeEmbedding(nodeId: string)` — обновление embedding для узла
   - Добавить метод `searchMemory(query: string, topK?: number): Promise<MemorySearchResult[]>`

3. Формирование текста для векторизации:
   ```typescript
   function buildEmbeddingText(node: GraphNode): string {
     const parts = [node.name];
     if (node.path) parts.push(`File: ${node.path}`);
     if (node.metadata?.why) parts.push(`Why: ${node.metadata.why}`);
     if (node.metadata?.how_to_apply) parts.push(`How to apply: ${node.metadata.how_to_apply}`);
     if (node.metadata?.description) parts.push(node.metadata.description);
     return parts.join('. ');
   }
   ```
