# Архитектурные гайдлайны

## Принципы
- SOLID — разделение ответственности
- DRY — не повторять код
- KISS — простота превыше всего
- YAGNI — не делать то, что не нужно сейчас

## Структура проекта
src/
├── components/ # UI-компоненты (React/Vue)
├── services/ # Бизнес-логика
├── utils/ # Утилиты и хелперы
├── interfaces/ # TypeScript интерфейсы
└── types/ # TypeScript типы

## Именование
- Файлы: PascalCase для компонентов, camelCase для утилит
- Переменные: camelCase
- Константы: UPPER_SNAKE_CASE
- Интерфейсы: I + PascalCase (например, IUserService)

## Обработка ошибок
- Всегда использовать try/catch для асинхронных операций
- Логировать ошибки через logger
- Показывать пользователю понятные сообщения

## Тестирование
- Покрытие кода >70%
- Unit-тесты для всех сервисов
- Integration-тесты для критичных сценариев
