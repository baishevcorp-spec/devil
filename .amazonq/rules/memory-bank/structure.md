# Devil — Структура проекта

Описание организации кода и архитектуры расширения.

## Директории проекта

### src/
Основная директория с исходным кодом расширения.

- **commands/** — обработчики команд, доступных в VS Code и через чат
- **interfaces/** — TypeScript-интерфейсы и типы для сервисов
- **panels/** — панели интерфейса (ChatPanel — чат-интерфейс)
- **providers/** — провайдеры данных и компонентов
- **services/** — основные сервисы (LLM, память, поиск, файловая система)
- **utils/** — утилиты (логгер, обработка ошибок)
- **extension.ts** — точка входа в расширение

### tests/
Тесты расширения.

- **unit/** — модульные тесты для отдельных компонентов
- **integration/** — интеграционные тесты
- **__mocks__/** — мок-объекты для тестирования

### webview/
Файлы веб-интерфейса чата.

- **chat.html** — HTML-шаблон
- **chat.css** — стили
- **chat.js** — логика взаимодействия с контекстом

### .devil/
Исторические данные и настройки расширения.

- **checklist.md** — чек-лист задач
- **history.json** — история действий
- **roadmap.md** — план развития

### docs/
Документация проекта.

- **instructions/** — инструкции и FAQ
- **roadmap/** — планы развития (Roadmap, checkpoint, TZ)
- **ARCHITECTURE.md** — архитектурное описание
- **DATABASE_SCHEMA.md** — схема базы данных графа

## Основные компоненты

### Расширение (extension.ts)
Точка входа в расширение. Регистрирует команды, создаёт панель чата и управляет жизненным циклом.

### Панель чата (ChatPanel.ts)
Реализация WebView-панели с чат-интерфейсом. Управляет сообщениями, командами, отправкой запросов к LLM и обработкой ответов.

### Сервис LLM (LLMProvider.ts)
Интеграция с OpenAI-совместимыми API. Отправка запросов, обработка ответов, управление контекстом.

### Графовая память (MemoryStore.ts, GraphBuilder.ts)
Хранение информации о проекте в виде графа узлов (классы, функции, файлы, типы) с рёбрами зависимостей.

### Управление проектом (ProjectManager.ts)
Сканирование файлов, построение графа, управление состоянием проекта.

### Поиск (SearchIndex.ts)
Индексация и поиск по содержимому файлов с поддержкой регулярных выражений.

### Git-сервис (GitService.ts)
Интеграция с Git: diff, история коммитов, информация о файлах.

### Контекст и история (ContextBuilder.ts, HistoryManager.ts)
Формирование контекста для LLM на основе истории чата и графа проекта.

### Конфигурация (ConfigManager.ts)
Управление настройками расширения (API-ключ, URL, модель, путь к проекту).

## Архитектурные паттерны

- **Service Layer** — каждый сервис инкапсулирует определённую функциональность
- **Command Pattern** — команды выражаются через обработчики с единым интерфейсом
- **Dependency Injection** — сервисы передаются через конструкторы
- **TypeScript Interfaces** — чёткое разделение интерфейсов и реализаций

3. **Поиск → SearchIndex → Результаты → ChatPanel**
